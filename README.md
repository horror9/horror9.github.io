📘 フィクション

「勤怠アプリ、大混乱 — 小さな指の業務停止力」

康平（35）は、ノートPCで会社の勤怠アプリを開き、
今日の出勤打刻や残業申請を確認していた。

画面には：
	•	打刻ボタン
	•	休暇申請フォーム
	•	残業入力フォーム
	•	各種承認ステータス

すべて、マウス一つで操作可能。

リビングのソファには、3歳の息子・りく。
好奇心と破壊力を兼ね備えた小さな手が、PCに届く距離にある。

⸻

🌀 第一幕：無邪気なタッチ連打

康平がキッチンに立つと、りくが椅子に登ってキーボードを触り始める。
	•	「打刻」ボタンを 何度も連打
	•	「残業申請」フォームに ランダムな数字 を入力
	•	「休暇申請」ボタンを ポチポチ連打

画面には赤と青の通知がチカチカ：

出勤打刻済み
残業申請送信
休暇申請エラー

りく
「ぴっぴっ、ボタンたのしい〜！」

⸻

🌀 第二幕：会社サーバーが悲鳴

康平が戻ると、勤怠アプリの画面は完全に混乱状態。
	•	打刻ボタンは 連打で重複
	•	残業申請が 999時間 とかに化けている
	•	休暇申請は 承認待ちステータスが無限ループ
	•	同僚から Slack に問い合わせが飛んでくる

康平
「え、ちょ、これ…俺のせい…じゃないよね……？」

りく
「いっぱい押したよ〜！」

サーバー側ログは真っ赤。
データベースには ありえない残業時間と休暇申請 が散乱していた。

⸻

🌀 第三幕：緊急復旧と再発防止

康平は冷静に対応：
	1.	勤怠アプリを一時停止
	2.	データベースをバックアップから復旧
	3.	会社の総務に事情を説明して混乱を最小限に
	4.	ノートPCは 物理的に子どもが触れない場所 に設置

さらに、心の中で誓う：
	•	勤怠アプリは仕事用PCでのみ操作
	•	子どもには 絶対触らせない
	•	ログイン情報は常に手元に保持

りくは笑顔で言った。
「ボタン、いっぱい押したよ〜！」

康平
「その笑顔が……憎い……！」

⸻

🧩 教訓
	•	会社の勤怠アプリは子どものおもちゃではない
	•	打刻や申請は1回1回慎重に、二重登録や誤入力のリスクを理解する
	•	物理的なアクセス制限が最強のセキュリティ
	•	子どもは「ボタンを押すだけ」で業務に大ダメージを与える存在

康平はその夜、PCの前で小さな指を見ながら思った。

「子どもは天使……でも業務停止の魔王……」
